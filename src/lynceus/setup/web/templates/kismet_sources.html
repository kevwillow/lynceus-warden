Kismet is reachable but reports no Wi-Fi datasource.

Common causes:

  • No Wi-Fi adapter installed on the Kismet host.
  • Adapter not in monitor mode (Kismet refused to start the source).
  • Kismet user lacks the right group membership (typically the kismet group).

Fix the underlying Kismet config (typically /etc/kismet/kismet_site.conf), e.g. source=wlan1:name=external_wifi, then restart Kismet and re-run this wizard. sudo lynceus-bootstrap-kismet --reset-config can rewrite Kismet's config in place if you'd rather start from a known-good baseline.

The wizard can't continue from this page until Kismet has at least one Wi-Fi source — pick a path: Previous to retry the probe after fixing Kismet, or Cancel to exit the wizard and come back after.
