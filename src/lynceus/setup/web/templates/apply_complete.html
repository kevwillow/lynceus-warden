{% if scope == "system" %}

Production (systemd, persistent):
sudo systemctl enable --now lynceus.service lynceus-ui.service

Enables and starts both the poller and the web UI as system services. They will auto-restart on failure and persist across reboots. To watch initial poll output: sudo journalctl -u lynceus.service -f.

Optional — weekly Argus watchlist refresh:
sudo systemctl enable --now lynceus-refresh.timer

Off by default: this is the only Lynceus surface that opts the host into a recurring outbound network call.

If you haven't configured Kismet for capture yet: run sudo lynceus-bootstrap-kismet --skip-install now. It puts your Wi-Fi adapter into monitor mode, adds the source= lines to /etc/kismet/kismet_site.conf, and adds your user to the kismet group. This is required for lynceus to see any devices — even after this wizard completes successfully. The --skip-install flag tells it Kismet is already installed (which it is, or you wouldn't be on this page).
